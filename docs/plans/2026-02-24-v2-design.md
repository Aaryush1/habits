## Workstream 2: Notifications

### Per-Habit Reminders

- Each habit gets an optional `reminderTime` field (nullable, stored as minutes-since-midnight int for Hive simplicity).
- **Auto-suggest from implementation intentions**: If the user filled in the "When?" field, pre-fill the reminder time.
- **Notification text**: Habit name + identity reinforcement. E.g., "Time to Meditate â€” You're becoming a calm, focused person." Pulls from identity statement if set, falls back to habit name only.
- Fires only on days the habit is scheduled (respects weekly/monthly schedule).
- **Quick-complete action**: Android action button to mark done without opening the app. Tap notification body â†’ opens detail screen.

### Evening Nudges

- Single configurable time (default 9pm), stored in Settings.
- Checks remaining uncompleted scheduled habits. If > 0: "You still have 3 habits left today."
- If all done and `positiveNudgeEnabled`: "All done! Perfect day."
- Skips if everything completed before nudge time.

### Permission Flow

- **Lazy request**: Don't ask on first launch. Ask when user first enables a reminder or toggles nudges.
- If denied: inline message explaining how to enable in system settings. No nagging.
- Uses `flutter_local_notifications` + `permission_handler` (both already in pubspec).

### Data Model Changes

- `Habit` entity: add `reminderTime` (int?, HiveField 19) â€” minutes since midnight, nullable
- New Hive box `settingsBox` with keys:
  - `eveningNudgeEnabled` (bool, default false)
  - `eveningNudgeTime` (int, default 1260 = 9pm)
  - `positiveNudgeEnabled` (bool, default true)

### Notification Scheduling

- On app launch + after any habit edit: cancel all, reschedule all via `zonedSchedule`.
- Habit notification ID: `habitId.hashCode.abs() % 100000` (unique per habit, avoids collisions).
- Evening nudge ID: fixed 0.
- Cancel + reschedule pattern (simpler than diffing).

## Workstream 3: Habit Stacks

### Data Model

- New `Stack` entity: `id`, `name`, `description`, `habitIds` (ordered List<String>), `iconEmoji` (String?), `createdAt`.
- New Hive box `stacksBox` with `StackModel` adapter.
- Many-to-many: a habit can belong to multiple stacks (stack holds habit IDs, habits don't reference stacks).
- **Templates**: Preset stacks shipped with app â€” "Morning Routine", "Wind Down", "Workout Prep". Pre-filled Stack objects, customizable/deletable.

### Stack Builder UI

- **Stacks list screen**: Accessible from Habits tab via chip toggle at top (flat list vs. stacks view). Not a 5th tab.
- **Stack detail screen**: Habits shown as a connected vertical chain â€” line connecting circles/cards. Each card: habit name, color dot, today's completion status. Drag handles via `ReorderableListView`.
- **Create/edit stack flow**: Bottom sheet. Name + optional description + multi-select habit list with drag-to-reorder. "Start from template" chips at top.
- **Delete stack**: Swipe-to-delete or long-press menu. Deleting a stack doesn't delete habits â€” just the grouping.

### Triggered Chain Flow

- Completing habit A in a stack shows snackbar: "Next up: Habit B" with "Start" action button.
- If notifications enabled, completing a habit can fire notification for next in stack (configurable per-stack: "Chain notifications" toggle).
- Chain prompt only shows when completing a habit that belongs to a stack and the next habit is uncompleted today.

### Stack Analytics

- **Completion funnel**: Horizontal bars, widest at top (first habit), narrowing per drop-off. Each bar in the habit's own color.
- **Stack completion rate**: % of days where ALL habits in the stack were completed.
- **Weakest link**: Which habit in the stack has the lowest individual completion rate â€” the bottleneck.
- Accessible from stack detail screen as a collapsible section at bottom.

## Visual Design Direction

### Design Language (carried from v1)

- **Dark-first**: `backgroundPrimary` (#0F0F0F) base, layered cards in `backgroundSecondary`/`Tertiary`
- **Accent system**: Gold for primary actions, Green for completion/success, Coral for missed/errors, Blue (`twoMinuteBlue`) for effort/time
- **Typography**: Google Fonts + JetBrains Mono for numeric data
- **Cards**: 10-14px radius, subtle `borderSubtle` border, depth via layered backgrounds (not shadows)
- **Spacing**: Consistent via `AppSpacing` constants

### New Components

| Component | Used In | Visual Direction |
|---|---|---|
| Empty state illustration | Home, Analytics, Stacks | Phosphor line-art icon + text + CTA. Muted colors |
| Chain connector | Stack detail | Vertical line with circles at each habit. Habit's color when complete, `backgroundQuaternary` when pending |
| Completion particle burst | Home toggle | 4-6 dots in habit's color, radiate ~20px, fade 300ms |
| Milestone celebration | Home (streak milestones) | Gold confetti, centered, 1.5s. Toast with streak count |
| Notification settings tile | Settings, Habit form | Time picker chip in `twoMinuteBlue`, consistent with duration chips |
| Stack funnel chart | Stack analytics | Horizontal bars, each habit's color, narrowing per drop-off |
| Template picker | Stack form | Horizontal scroll cards with emoji + name, like color picker pattern |

### Transition Specs

- Tab switch: 300ms fade-through
- Card â†’ detail: Hero 350ms
- Analytics card â†’ drill-down: Container transform 400ms
- Sheet entrance: Scale 0.95â†’1.0 + fade, 250ms
- Completion toggle: Existing bounce + particle burst at 200ms offset

### Color Usage for New Features

- **Stacks**: Inherit first habit's color for header gradient
- **Notifications**: `twoMinuteBlue` for all notification UI (time chips, nudge toggle)
- **Empty states**: `textTertiary` text, `backgroundTertiary` illustration bg, `accentGold` CTA
- **Milestones**: Gold confetti, `accentGoldSubtle` toast background
